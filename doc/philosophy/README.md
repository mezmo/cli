## Philosophy

These are what we consider to be the fundamental principles of good CLI design.

### Core Principles

| Principle                                                       | Description                                                          |
|-----------------------------------------------------------------|----------------------------------------------------------------------|
| [Chaos](./chaos.md)                                             | Embracing and managing the unpredictable nature of user interactions |
| [Consistency Across Commands](./consistency-across-commands.md) | Maintaining uniform behavior and patterns throughout the CLI         |
| [Conversation As Norm](./conversation-as-norm.md)               | Treating CLI interactions as natural conversations with users        |
| [Ease Of Discovery](./ease-of-discovery.md)                     | Making features and commands easily discoverable by users            |
| [Empathy](./empathy.md)                                         | Understanding and addressing user needs and frustrations             |
| [Human First Design](./human-first-design.md)                   | Prioritizing human experience over technical implementation          |
| [Robustness](./robustness.md)                                   | Building resilient systems that handle errors gracefully             |
| [Saying Just Enough](./saying-just-enough.md)                   | Providing the right amount of information at the right time          |
| [Simple Parts That Work Together](./simple-parts-work-together.md)   | Creating modular components that integrate seamlessly                |
